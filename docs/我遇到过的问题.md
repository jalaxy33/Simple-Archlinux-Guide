# 我遇到过的问题

## 安装 usb 无线网卡

主要参考 [archwiki - 网络配置/无线网络配置](https://wiki.archlinuxcn.org/wiki/网络配置/无线网络配置)，不保证所有网卡都能成功。

1. 首先安装 `usbutils` 工具：

   ```sh
   sudo pacman -S --needed usbutils
   ```

   这个工具提供了 `lsusb` 命令

2. 执行 `lsusb` 命令查看网卡信息

   ```sh
   lsusb
   ```

   以我的绿联 CM762 网卡为例，显示：

   ```sh
   # 其他设备信息
   # ...

   # 网卡信息
   Bus 001 Device 009: ID 368b:8d85 AICSemi AIC 8800D80
   ```

   设备 ID 就是 `368b:8d85`，其中 `368b` 是制造商编号，`8d85` 是产品编号

   同时可以看到芯片的型号是 `AIC 8800D80`

3. 在 AUR 中搜索网卡驱动后安装：

   ```sh
   yay -S aic8800d80-dkms  # 改为自己的网卡驱动
   ```

4. 重启

   ```sh
   reboot
   ```

## 为挂载目录设置读写权限

其他分区的挂载目录默认只为 root 所有，为了能让其他具有 sudo 权限的用户也能自由的访问这些目录，可以修改这些挂载目录的读写权限。

1. 确保用户属于目标组。arch的sudo权限组是 `wheel`，其他发行版可能是 `sudo`。

   ```sh
   # 查看当前用户的组
   groups

   # 将用户添加到 wheel 组（如果需要）
   sudo usermod -aG wheel $USER

   # 查看 wheel 组的 GID
   getent group wheel
   # 输出示例：wheel:x:998:user1,user2
   ```

2. 修改目录所属用户组

   修改为 `wheel` 组，其他发行版可能是 `sudo` 组

   ```sh
   sudo chown -R :wheel /path/to/directory
   ```

   `-R` 表示递归修改

3. 修改目录读写权限

   ```sh
   sudo chmod -R 2775 /path/to/directory
   ```

   - 开头的2是setgid 位，确保新创建的文件和目录继承父目录的组

   - 775表示对所属用户和组可读写，其他用户只读（不希望其他用户访问设为770）

## 为应用设置开机自启

有些应用不提供这个选项，只要在 `/usr/share/applications` 目录下找到软件的 `.desktop` 文件，将其复制到 `~/.config/autostart/` 目录下即可。

如果要取消自启，把这个文件从`~/.config/autostart/` 目录中移除即可。

## Vim与系统剪贴板同步

1. 安装 `gvim`，这是支持剪贴板的图形界面版本的 vim：

   ```sh
   sudo pacman -S --needed gvim
   ```

   会提示与原有的 vim 冲突，选 y 确定卸载原有 vim。

2. 设置与系统剪贴板同步

   ```
   vim ~/.vimrc
   ```

   ```sh
   "与系统剪贴板同步
   set clipboard=unnamedplus
   ```

## 配置SSH服务

如果需要让其他设备通过 SSH 连接到本机，需要配置 SSH 服务。

1. 安装 `openssh`：

    ```sh
    sudo pacman -S --needed openssh
    ```

2. 编辑 SSH 配置文件：

    ```sh
    sudo vim /etc/ssh/sshd_config
    ```

    ```
    # 取消下面几行的注释
    AllowAgentForwarding yes
    AllowTcpForwarding yes
    TCPKeppAlive yes
    PasswordAuthentication yes

    # 取消注释，将值改为yes
    PermitRootLogin yes
    ```

3. 启动 SSH 服务，并设为开机启动：

    ```sh
    sudo systemctl enable --now sshd
    ```
