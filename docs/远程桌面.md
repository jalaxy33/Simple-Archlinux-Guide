# 远程桌面

可以用 `parsec` 或者 `sunshine+moonlight` 日月组合，推荐后者，性能更好。

## parsec

需要先注册账号。两台设备都安装 parsec，archlinux上通过 AUR 安装：

```sh
yay -S parsec-bin
```

之后在两台设备上开启软件，登录同一个账号，就可以远程串流了。

## sunshine+moonlight

参考资料：[在Arch Linux上使用Sunshine和Moonlight实现使用平板电脑作为副屏](https://www.rainafter.cn/posts/在arch-linux上使用sunshine和moonlight实现使用平板电脑作为副屏/) | [Linux上利用sunshine串流和虚拟显示器实现副屏扩展](https://www.cnblogs.com/twinblade-i/p/18797195)

### 1. 创建虚拟显示器

正常情况下，使用远程桌面必须连接显示器，但是可以通过一些方法进行伪装。

配置虚拟显示器的方式分为两类：一种是已有显卡欺骗器硬件的方式，另一种是通过创建虚拟显示器的方式。前者只要在电脑上插入显卡即可解决问题，如果没有显卡欺骗器，可以参考以下步骤创建虚拟显示器。

> 即使已有显卡欺骗器硬件，也依然建议参考后面的部分通过修改EDID文件将显示器分辨率修改为目标显示设备的分辨率，可以极大的提升使用体验。

在Wayland环境下，类似于使用XrandR工具等大多数创建虚拟显示器的方法均已经失效，本文使用了[内核级显示模式（KMS）](https://wiki.archlinuxcn.org/wiki/内核级显示模式设置)来创建虚拟显示器器。

1. 查看当前系统记录的显示器设备接口的连接情况，在`bash`或`zsh`使用命令
   ```sh
   for p in /sys/class/drm/*/status; do con=${p%/status}; echo -n "${con#*/card?-}: "; cat $p; done
   ```
   选取一个标记为`disconnected`的空闲接口（如`HDMI-A-1`）
2. 编辑GRUB配置文件，在 `GRUB_CMDLINE_LINUX_DEFAULT` 之后的引号中填入接口参数：
   ```sh
   sudo vim /etc/default/grub
   ```
   ```sh
   # 注意将 HDMI-A-1 改成自己的接口
   GRUB_CMDLINE_LINUX_DEFAULT="...  video=HDMI-A-1:e "
   ```
3. 保存退出，更新 GRUB 配置：

```sh
sudo grub-mkconfig -o /efi/grub/grub.cfg   # 我的 grub 在/efi下，一般的可能是在 /grub 下
```

4. 重启系统后在系统的显示（Display）设置应该就能看到一个虚拟显示器。

### 2. 创建EDID配置文件

推荐是导出现有显示器 edid 配置文件，在此基础上对分辨率等参数进行修改：

1. 首先找到系统连接到的虚拟显示器

   ```sh
   # 找出您的虚拟显示器路径
   find /sys/devices/ -name edid -type f
   ```

   此处会列举出很多显示器，在这里可以根据显示器所连接到的显卡来判断哪一个是虚拟显示器。

   > 如果在这一步不确定哪一个是新添加的虚拟显示器的EDID文件，也可以导出任意一个显示器的EDID文件（例如笔记本电脑的内置屏幕），然后在后续的步骤中自定义EDID文件修改显示器的分辨率即可。（经测试基本上没有任何影响）

2. 再将虚拟显示器的EDID数据复制到一个想要保存的目录中

   ```sh
   # 复制 EDID 数据 (替换路径为上面找到的实际路径)
   sudo cp /sys/devices/pci0000:00/0000:00:02.0/drm/card0/card1-HDMI-A-3/edid /xxxx想要保存到的路径xxxx/edid.bin
   ```

3. 自定义 EDID

   在这里推荐一个好用的EDID在线编辑网站[EDID解析与编辑工具(点击可查看)](https://edid.wherelse.cc/)，我们在网站左侧上传刚才保存的EDID文件，点击解析后，在网站下面的水平和垂直分辨率部分修改为目标设备的原生分辨率。在这里也可以也改一下显示器名称，以示区别。

   自定义EDID文件 编辑完成后，点击底部的下载EDID文件，并保存到一个目录中。

4. 将刚才下载的自定义EDID文件复制到系统中

   可能需要先创建一下目录：

   ```sh
   sudo mkdir -p /usr/lib/firmware/edid
   ```

   ```sh
   sudo cp /xxxx下载的自定义EDID文件的路径xxxx/edid.bin /usr/lib/firmware/edid/my-edid.bin
   ```

5. 然后修改GRUB配置文件
   ```sh
   sudo vim /etc/default/grub
   ```
   ```sh
   # 注意将 HDMI-A-1 改成自己的接口，my-edid.bin 修改成自己的文件名
   GRUB_CMDLINE_LINUX_DEFAULT="...  drm.edid_firmware=HDMI-A-1:edid/my-edid.bin "
   ```
6. 保存退出，更新 GRUB 配置：

   ```sh
   sudo grub-mkconfig -o /efi/grub/grub.cfg   # 我的 grub 在/efi下，一般的可能是在 /grub 下
   ```

7. 重启后，即可看到虚拟显示器的分辨率已经变为我们指定的分辨率。

后续将新添加的虚拟显示器作为 sunshine 的默认捕捉显示器即可。

### 3. 安装 sunshine 服务端软件

- 安装

  可以使用Extra软件源中的正式版的 `sunshine`，也可以使用AUR软件源中的预发布版 `sunshine-beta-bin`

  ```bash
  sudo pacman -S --needed sunshine
  ```

  ```bash
  yay -S sunshine-beta-bin
  ```

  两者二选一即可，后者更新一点。

- 使用

  打开软件后会出现一个浏览器页面（`https://localhost:47990/`），第一次要你创建用户名和密码
  1. 在 sunshine WebUI `Troubleshooting # Logs` 中找到`KMS Monitor List`，从中获取虚拟显示器接口对应的编号，将 sunshine WebUI 中`Configuration # Audio/Video # Display number` 设置为对应编号

  2. 对Configuration中其他选项按需调整。推荐将 「Input」中的「将右Alt 键映射到 Windows 键」 打开。

  3. 保存并应用 sunshine 配置更改

  4. 查看本机 ip 地址
     ```sh
     ip addr show
     #或者
     ip a
     ```

### 4. 安装 moonlight 客户端

在其他设备上安装 moonlight，Arch 上可以通过 pacman 或者 flatpak 安装

```sh
sudo pacman -S --needed moonlight-qt
```

```sh
flatpak install com.moonlight_stream.Moonlight
```

两者二选一即可。

通过 ip 地址搜索设备，第一次连接会提示一个 pin 码，在服务端subshine网页里填写这个 pin 码和客户端的设备名称，然后应该就能远程串流了。

### moonlight快捷键

```
Ctrl + Alt + Shift + Q: 退出串流，但游戏进程在服务器端仍然运行。
Ctrl + Alt + Shift + X: 切换全屏模式和窗口模式。
Ctrl + Alt + Shift + S: 开启或关闭性能统计信息叠加显示 (如 FPS, 延迟等)。
Ctrl + Alt + Shift + Z: 捕获/释放鼠标，方便在游戏和桌面之间切换。
```
