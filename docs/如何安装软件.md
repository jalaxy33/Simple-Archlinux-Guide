# 如何安装软件

参考资料：[软件安装相关](https://github.com/SHORiN-KiWATA/ShorinArchExperience-ArchlinuxGuide/wiki/软件安装相关)

## 安装软件的几种主要方式：

安装软件的几种主要方式：

1. `pacman`从官方仓库和 archlinuxcn 下载软件
2. `yay`/`paru` 从 [AUR](https://wiki.archlinuxcn.org/zh-sg/Arch_用户软件仓库)（archlinux用户仓库）下载软件
3. `flatpak` 从 [flathub](https://flathub.org/en)或指定的flatpak源下载
4. 应用官网下载 appimage 包（免安装便携版）直接运行
5. `homebrew` 安装
6. `wine`/`proton` 兼容层转译 windows 应用
7. 在 `amber-ce` 容器中使用「星火应用商店」安装

建议优先级：pacman >> flatpak > aur > 官网下载appimage > homebrew 安装 > 兼容层转译 ≈ 星火应用商店

## 1. 添加 archlinuxcn 源

[archlinuxcn源](https://www.archlinuxcn.org/archlinux-cn-repo-and-mirror/)包含了很多好用的软件。

1. 编辑pacman配置文件添加archlinuxcn源

   ```bash
   sudo vim /etc/pacman.conf
   ```

2. 按 G 到文件底部，写入（ctrl+shift+V粘贴）

   ```bash
   [archlinuxcn]
   Server = https://mirrors.cloud.tencent.com/archlinuxcn/$arch
   Server = https://mirrors.ustc.edu.cn/archlinuxcn/$arch
   Server = https://mirrors.tuna.tsinghua.edu.cn/archlinuxcn/$arch
   Server = https://mirrors.hit.edu.cn/archlinuxcn/$arch
   ```

   更多源可以在[这里](https://github.com/archlinuxcn/mirrorlist-repo)找到。

3. 同步数据库并安装archlinuxcn密钥

   ```bash
   sudo pacman -Sy archlinuxcn-keyring
   ```

## 2. 安装 AUR 助手

用于从 [AUR](https://wiki.archlinuxcn.org/zh-sg/Arch_用户软件仓库) 用户软件仓库下载软件。

> 注意：aur很方便，但要注意辨识它是否安全。

### yay

如果添加了 archlinuxcn 源，可以直接 pacman 安装 yay：

```bash
sudo pacman -S yay
```

### paru

paru 是另一个 AUR 助手，两个都可以安装，不冲突。

```bash
sudo pacman -S paru
```

## 3. 安装 flatpak

这是种“一次打包全发行版通用”的打包方式，浏览器搜索flathub可以获取软件的安装链接，安装工具是 [flatpak](https://wiki.archlinuxcn.org/wiki/Flatpak)。如果需要图形界面，GNOME 的flatpak商店叫 `gnome-software`，KDE 的叫 `discover`。

`flatpak` 从 [flathub](https://flathub.org/en) 或指定的flatpak源下载软件。

### 安装 flatpak

```sh
sudo pacman -S --needed flatpak
```

图形界面，可以在装桌面环境的时候再安装

- GNOME 桌面
  ```bash
  sudo pacman -S --needed gnome-software
  ```
- KDE 桌面
  ```bash
  sudo pacman -S --needed discover
  ```

### 添加 flathub 官方源

安装完后一般已经添加了 flathub 官方源，如果没有可以手动添加：

```bash
flatpak remote-add --user --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo
```

### 更换国内源

注意：即使用了镜像站 你也需要让你的网络能访问flathub官方仓库 你才能下载所有的包

- 上交大
  ```bash
  sudo flatpak remote-modify flathub --url=https://mirror.sjtu.edu.cn/flathub
  ```
  如果出现错误可尝试：
  ```bash
  wget https://mirror.sjtu.edu.cn/flathub/flathub.gpg
  sudo flatpak remote-modify --gpg-import=flathub.gpg flathub
  ```
- 中科大
  ```bash
  flatpak remote-modify flathub --url=https://mirrors.ustc.edu.cn/flathub
  ```

## 3. 如何使用 appimage

appimage和flatpak类似，也是所有发行版通用的打包方式，可以理解为免安装便携版，下载后可以直接执行。

appimage 来源：

- 软件官网
- [appimagehub](https://www.appimagehub.com/browse?ord=latest)这个网址有很多有趣的appimage应用，有兴趣的可以搜索玩玩看

### 直接执行

需要安装`fuse3`和`fuse2`，通常已经安装好了。文件下载下来之后右键属性设置可执行权限之后即可运行，或者使用命令：

```bash
chmod +x ~/path/to/files.appimage
```

感兴趣的可以使用这条命令把appimage解压出来看看里面都有什么：

```bash
/path/to/files.appimgae --appimage-extract
```

会出现一个`squashfs-root`文件夹，里面就是解压出来的文件。

### 把appimage集成到系统

- 方法一：gearlever（推荐）

  flathub下载

  ```bash
  flatpak install flathub it.mijorus.gearlever
  ```

  或者 aur 下载

  ```bash
  yay -S --needed gearlever
  ```

  由于appimage的安装涉及到解压和打包，所以用gearlever打开较大的appimage会有一点慢，稍等一会就会出现启动或者集成到系统的选项了。卸载也是用这个软件。

- 方法二：appimagelauncher

  ```bash
  yay -S --needed appimagelauncher
  ```

  - **安装appimage**：安装后启动appimage时会弹出appimagelauncher的窗口，第一次启动会让你设置安装路径，然后让你选择运行一次还是集成到系统（有时候会安装失败或者安装之后无法运行）。
  - **卸载appimage**：右键快捷方式，点击remove appimage from system，或者手动删除~/.local/share/Applications下的destop文件和安装目录下的appimage文件。

## 4. 安装homebrew（不推荐）

[Homebrew](https://brew.sh/) 为很多软件提供了跨发行版的安装管理方式，也是 MacOS 事实上的包管理系统。在 Linux 上，homebrew 会创建一个具有 sudo 权限的 `linuxbrew` 用户，并将软件安装到该用户目录下，若要使用由 homebrew 安装的软件，需要先激活 brew 或通过软链接到相应路径。

绝大部分软件都可以通过前几种方式安装，因此不做推荐。

> **另一个原因**：homebrew 在多用户的情况下，只有安装 homebrew 的用户具有 `/home/linuxbrew` 的写权限，因此只有该用户具有管理软件的权限。

### 安装方式一：直接安装

直接用 homebrew 官方安装脚本安装，需要能访问 github 的网络环境：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 安装方式二：镜像源安装

利用中科大源安装：

- bash/zsh：
  设置环境变量
  ```bash
  export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.ustc.edu.cn/brew.git"
  export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.ustc.edu.cn/homebrew-core.git"
  export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.ustc.edu.cn/homebrew-bottles"
  export HOMEBREW_API_DOMAIN="https://mirrors.ustc.edu.cn/homebrew-bottles/api"
  ```
  之后在命令行运行 Homebrew 安装脚本：
  ```bash
  /bin/bash -c "$(curl -fsSL https://mirrors.ustc.edu.cn/misc/brew-install.sh)"
  ```
- fish：
  ```fish
  set -x HOMEBREW_BREW_GIT_REMOTE "https://mirrors.ustc.edu.cn/brew.git"
  set -x HOMEBREW_CORE_GIT_REMOTE "https://mirrors.ustc.edu.cn/homebrew-core.git"
  set -x HOMEBREW_BOTTLE_DOMAIN "https://mirrors.ustc.edu.cn/homebrew-bottles"
  set -x HOMEBREW_API_DOMAIN "https://mirrors.ustc.edu.cn/homebrew-bottles/api"
  ```
  ```fish
  /bin/bash -c "$(curl -fsSL https://mirrors.ustc.edu.cn/misc/brew-install.sh)"
  ```

这些环境变量在以后安装更新软件时也能用到，建议写入对应的 shell 配置文件中，如 `~/.bashrc`，`~/.zshrc`，`~/.config/fish/config.fish`

### 激活 brew 命令

```bash
eval "$(/home/linuxbrew/.linuxbrew/bin/brew shellenv)"
```

可以将这个命令添加到所使用 shell 的配置文件中，如 `~/.bashrc`，`~/.zshrc`，`~/.config/fish/config.fish`，重启 shell。

然后就可以使用 brew 命令了：

```bash
brew
```

## 5. 星火应用商店（不推荐）

[shorin - 安装星火应用商店](https://github.com/SHORiN-KiWATA/ShorinArchExperience-ArchlinuxGuide/wiki/软件安装相关#星火应用商店)

[星火应用商店](https://www.spark-app.store/) 是支持 debain 系 linux 发行版的社区应用商店，定位类似于 Arch 下的 AUR。ArchLinux 可以通过 [amber-ce](https://gitee.com/amber-ce) 容器来安装 deb 包。

由于是运行在 debian 容器中的，可能会有各种小问题，不是很推荐，有需要的可以创建快照后尝试一下。

已知问题：

- fish下无法正常生成快捷方式，想用星火商店的话后续更换shell的部分使用zsh。
- ibus输入法只能在gtk应用中使用，qt应用中无法使用；

### 安装 ace 容器和星火应用商店

1. 安装 amber-ce 容器
   ```bash
   # 安装 debian 13 容器，debian 12 是 amber-ce-bookworm
   yay -S --needed amber-ce-trixie
   ```
   安装完成后没有显示图标的话登出一次
2. 官网下载星火应用商店的deb包：[官网](https://www.spark-app.store/download)
3. 打开蓝色图标的 ace，里面是一个 debian 环境
4. 用 apt 安装星火应用商店
   ```bash
   sudo apt install /path/to/spark-store_x.x.x_amd64.deb
   ```
   指定安装包的绝对路径，可以手动输入，也可以把安装包拖拽进终端里自动输入路径

### ace 容器中启用输入法

- ibus
  仅gtk应用可用，解决办法暂时未知
- fcitx5
  ```bash
  sudo apt install fcitx5-frontend-gtk2 fcitx5-frontend-gtk3 fcitx5-frontend-gtk4 fcitx5-frontend-qt5 fcitx5-frontend-qt6
  ```

### 卸载 ace 容器中的软件

使用ace 软件卸载器卸载软件，或者 `sudo apt remove 【包名】`，或者直接删除整个容器，删除容器之后里面的所有东西都不会留下。

```bash
yay -Rns amber-ce-trixie
```
